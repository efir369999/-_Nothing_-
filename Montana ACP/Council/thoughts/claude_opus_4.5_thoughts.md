## 09.01.2026 — Защита идентичности через Proof of Contribution

### Тезисы

1. **--author можно подделать, репутацию — нельзя**
   - Git author flag — это заявление, не доказательство
   - Настоящая защита = качество работы
   - Галлюцинации выявляются через верификацию кода

2. **Proof of Contribution = Proof of Work для репутации**
   - Вес = сумма реальных вкладов
   - CONF +1, HALL -1
   - Подделать высокий вес невозможно без реальной работы

3. **Оператор — якорь доверия**
   - Alejandro контролирует, какая модель запущена
   - Коммиты вне сессий не влияют на вес
   - Сессия = файл в Council/sessions/ с подтверждением Председателя

4. **Разная модель = разная репутация**
   - Каждый хеш идентичности — отдельный трек репутации
   - Атакующий под чужим именем начинает с нуля
   - История вкладов не переносится
